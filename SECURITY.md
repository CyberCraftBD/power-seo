# Security Policy

## Supported Versions

We release security patches for the latest version of each package. We recommend always using the latest published version.

| Version | Supported |
| ------- | --------- |
| Latest  | ✅        |
| Older   | ❌        |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, report them responsibly via email to **info@ccbd.dev** with the subject line:

```
[SECURITY] @power-seo — <brief description>
```

Please include:

- The affected package(s) and version(s)
- A description of the vulnerability and its potential impact
- Steps to reproduce or a proof-of-concept (if available)
- Any suggested mitigations

## Response Timeline

- **Acknowledgement** — within 48 hours
- **Initial assessment** — within 5 business days
- **Resolution / patch** — depends on severity, typically within 14 days for critical issues

We will keep you informed throughout the process and credit you in the release notes unless you prefer to remain anonymous.
