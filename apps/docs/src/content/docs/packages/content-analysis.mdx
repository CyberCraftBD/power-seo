---
title: "@ccbd-seo/content-analysis"
description: "Yoast-style content scoring engine that analyzes titles, descriptions, keyphrase usage, headings, word count, images, and links."
---

## Install

```bash
npm install @ccbd-seo/content-analysis @ccbd-seo/core
```

## Quick Start

```typescript
import { analyzeContent } from '@ccbd-seo/content-analysis';

const result = analyzeContent({
  title: 'Best Running Shoes for Beginners',
  metaDescription: 'Discover the best running shoes for beginners with our expert guide.',
  keyphrase: 'running shoes for beginners',
  content: '<h1>Best Running Shoes</h1><p>Finding the right running shoes...</p>',
  url: 'https://example.com/best-running-shoes',
});

console.log(result.overallStatus); // "good" | "needs-improvement" | "poor"
console.log(result.results);
// [{ id: "title", status: "good", message: "..." }, ...]
```

## API Reference

### Entry Points

| Import | Description |
|---|---|
| `@ccbd-seo/content-analysis` | Core analyzer and individual check functions |
| `@ccbd-seo/content-analysis/react` | React components for analysis UI |

### Functions

| Export | Description |
|---|---|
| `analyzeContent(input)` | Runs all checks and returns `ContentAnalysisOutput` |
| `checkTitle(input)` | Checks title length, keyphrase presence, pixel width |
| `checkMetaDescription(input)` | Checks meta description length and keyphrase usage |
| `checkKeyphraseUsage(input)` | Checks keyphrase density and distribution |
| `checkHeadings(input)` | Validates heading hierarchy and keyphrase in H1 |
| `checkWordCount(input)` | Checks total word count against thresholds |
| `checkImages(input)` | Checks for images and alt text with keyphrase |
| `checkLinks(input)` | Checks for internal and external links |

### Types

| Type | Description |
|---|---|
| `ContentAnalysisInput` | Input object with title, description, keyphrase, content, url |
| `ContentAnalysisOutput` | Output with `overallStatus` and `results` array |
| `AnalysisResult` | Individual check result with `id`, `status`, `message` |
| `AnalysisStatus` | `"good"` \| `"needs-improvement"` \| `"poor"` |
