The `@power-seo` monorepo offers modular, tree-shakeable packages you can adopt incrementally — replace only what you need, one piece at a time.

## From next-seo

| next-seo | @power-seo | Notes |
|---|---|---|
| `<NextSeo>` | `<SEO>` from `@power-seo/react` | Same prop shape |
| `<DefaultSeo>` | `<DefaultSEO>` from `@power-seo/react` | Context-based defaults |
| `<ArticleJsonLd>` | From `@power-seo/schema/react` | Separate package |
| `useNextSeoConfig()` | `<DefaultSEO>` | Provider pattern |
| Next.js App Router metadata | `createMetadata()` from `@power-seo/meta` | Returns native `Metadata` object |

### Before

```typescript
import { NextSeo, ArticleJsonLd } from 'next-seo';

function Page() {
  return (
    <>
      <NextSeo title="My Page" description="Description" canonical="https://example.com" />
      <ArticleJsonLd headline="My Article" datePublished="2026-01-01" />
    </>
  );
}
```

### After

```typescript
import { SEO } from '@power-seo/react';
import { ArticleJsonLd } from '@power-seo/schema/react';

function Page() {
  return (
    <>
      <SEO title="My Page" description="Description" canonical="https://example.com" />
      <ArticleJsonLd headline="My Article" datePublished="2026-01-01" />
    </>
  );
}
```

For **Next.js App Router**, use `@power-seo/meta` for server-side metadata:

```typescript
import { createMetadata } from '@power-seo/meta';
export const metadata = createMetadata({ title: 'My Page', description: 'Description' });
```

## From react-helmet

| react-helmet | @power-seo/react | Notes |
|---|---|---|
| `<Helmet><title>` | `<SEO title="...">` | Prop-based API |
| `<Helmet><meta>` | `<SEO description="...">` | Structured props |
| `<Helmet><link rel="canonical">` | `<SEO canonical="...">` | Dedicated prop |
| `<Helmet><link rel="alternate">` | `<Hreflang alternates={[...]}>` | Typed hreflang component |

### Before

```typescript
import { Helmet } from 'react-helmet';
function Page() {
  return (
    <Helmet>
      <title>My Page | My Site</title>
      <meta name="description" content="Description" />
      <link rel="canonical" href="https://example.com" />
    </Helmet>
  );
}
```

### After

```typescript
import { SEO } from '@power-seo/react';
function Page() {
  return (
    <SEO title="My Page" titleTemplate="%s | My Site"
      description="Description" canonical="https://example.com" />
  );
}
```

## From Yoast (WordPress)

Moving from a WordPress + Yoast setup to a headless TypeScript stack? `@power-seo/content-analysis` is a direct Yoast-style replacement — same scoring checks, same green/orange/red feedback, framework-independent.

| Yoast Feature | @power-seo | Notes |
|---|---|---|
| SEO analysis panel | `analyzeContent()` from `@power-seo/content-analysis` | Returns score + per-check results |
| Readability analysis | `analyzeReadability()` from `@power-seo/readability` | Flesch-Kincaid, Gunning Fog, Coleman-Liau, ARI |
| SERP snippet preview | `generateSerpPreview()` from `@power-seo/preview` | Pixel-accurate at 580px |
| Focus keyphrase | `focusKeyphrase` field in `analyzeContent()` | Same concept, typed |
| Meta description editor | `validateMetaDescription()` from `@power-seo/core` | Char count + pixel-width validation |
| XML sitemaps | `generateSitemap()` from `@power-seo/sitemap` | Streaming, splitting, image sitemaps |
| Schema.org markup | `@power-seo/schema` | 23 builders + `validateSchema()` |
| Redirect manager | `@power-seo/redirects` | Next.js, Remix, Express adapters |

### Before (Yoast PHP)

```php
// Yoast handled automatically via WordPress hooks
add_filter('wpseo_metadesc', fn() => 'My meta description');
```

### After (TypeScript)

```typescript
import { analyzeContent } from '@power-seo/content-analysis';
import { analyzeReadability } from '@power-seo/readability';
import { generateSerpPreview } from '@power-seo/preview';

const content = analyzeContent({
  title: 'Best Coffee Shops in NYC',
  metaDescription: 'Discover the top coffee shops in New York City.',
  content: articleHtml,
  focusKeyphrase: 'coffee shops nyc',
  internalLinks: ['/blog', '/about'],
  externalLinks: ['https://maps.google.com'],
});

const readability = analyzeReadability({ content: articleHtml });
const serp = generateSerpPreview({ title, description: metaDescription, url, siteTitle });

console.log(content.score, '/', content.maxScore);  // e.g. 42 / 55
console.log(readability.status);                    // 'good' | 'ok' | 'poor'
console.log(serp.titleTruncated);                   // false
```

```tsx
// Real-time CMS editor feedback (React)

```
